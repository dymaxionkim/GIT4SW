GIT4SW - SolidWorks Git 버전 관리 클라이언트 사용자 가이드
==========================================================

GIT4SW는 SolidWorks 설계자를 위한 에이전틱 CAD 버전 관리 데스크톱 애플리케이션입니다. 표준 Git 워크플로우와 Git LFS(Large File Storage) 잠금 메커니즘을 통합하여 3D CAD 모델에서 동시 작업 시 덮어쓰기 없는 협업을 가능하게 합니다.

아래는 각 애플리케이션 탭, 위젯 및 기능에 대한 상세 설명입니다.

----------------------------------------------------------
1. 대시보드 모드 (DASHBOARD MODE)
----------------------------------------------------------
대시보드(Dashboard)는 애플리케이션의 중앙 컨트롤 패널로, 저장소 설정, 동기화 제어 및 실시간 상태를 표시합니다.

• 로컬 경로 (Local Path) (텍스트 필드 및 입력)
  - 활성 로컬 Git 저장소의 전체 경로를 표시합니다.
• 작업공간 변경 (Change Workspace) (버튼)
  - 폴더 선택 대화상자를 열어 다른 Git 저장소 디렉토리로 작업공간을 전환합니다.
• 원격 서버 (Remote Server) (텍스트 필드 및 입력)
  - 현재 작업공간에 연결된 원격 저장소 URL을 표시하거나, 새 저장소를 복제할 URL을 입력할 수 있습니다.
• 복제 (Clone) (버튼)
  - 원격 서버 URL의 Git 저장소를 로컬 경로에 지정된 디렉토리로 복제합니다.
• 활성 브랜치 (Active Branch) (콤보박스/드롭다운)
  - 현재 체크아웃된 브랜치를 표시합니다. 다른 브랜치를 선택하면 자동으로 해당 브랜치로 전환(체크아웃)합니다.
• 내 브랜치 만들기 (Make my branch) (버튼)
  - git/GitHub 설정에서 사용자 정보를 확인하여 사용자 이름 아래에 새 로컬 및 원격 개발자 브랜치를 자동으로 생성하고 전환합니다. 이렇게 하면 작업이 분리되어 'main' 브랜치에 대한 직접적인 수정 충돌을 방지합니다.
• README.md (버튼)
  - 활성 로컬 저장소의 README.md 파일을 메모장(Notepad)으로 엽니다. 텍스트 편집기를 편집하고 닫으면 클라이언트가 자동으로 git add, commit 및 push를 실행하여 파일을 원격 서버와 동기화합니다. 파일이 로컬에 없으면 템플릿에서 초기화합니다.
• 최신 버전 가져오기 (Get Latest Version (Sync)) (버튼)
  - 활성 브랜치의 최신 변경사항을 원격 서버에서 다운로드하여 적용(pull)합니다.
  - **충돌 자동 백업 (Conflict Auto-Backup)**: 동기화 중 병합 충돌이 감지되면, 클라이언트가 충돌 파일의 원본 로컬 버전을 작업공간 루트의 `.backup/` 폴더에 타임스탬프와 함께 자동으로 백업한 후(`[filename]_[timestamp].[ext]`) 충돌 해결 대화상자를 표시합니다. 실수로 덮어쓴 경우 로컬 수정사항을 쉽게 복원할 수 있습니다.
  - **최적화 검사 (Optimized Check)**: 사전 패치 커밋 비교를 수행합니다. 브랜치가 이미 원격 커밋과 최신 상태이거나 이미 완전히 병합된 경우 시간 및 시스템 오버헤드 절약을 위해 병합/pull을 건너뜁니다.
• main 브랜치를 현재 브랜치에 병합 (Merge main branch into current branch) (버튼)
  - 공유 'main'(또는 'master') 브랜치의 업데이트를 개발자 브랜치에 병합합니다. 자동으로 커밋하고, 병합된 브랜치를 원격 서버로 푸시한 후, 안전하게 작업공간을 원래 개발자 브랜치로 다시 전환합니다.
  - 병합 전 충돌 감지 시에도 로컬 변경사항을 보호하기 위해 동일한 자동 로컬 백업 프로세스가 실행됩니다.
  - **최적화 검사 (Optimized Check)**: 대상 브랜치의 커밋이 이미 병합되었거나 현재 브랜치 팁과 동일한지 확인하여 불필요한 병합 작업을 건너뜁니다.
• LFS 캐시 정리 (Cleanup LFS Cache) (버튼)
  - Git LFS 캐시 정리 마법사(Cleanup Wizard) 대화상자를 엽니다. 현재 인덱스와 최근 2개 커밋(`HEAD`, `HEAD~1`)에 포함된 파일을 제외한 `.git/lfs/objects/`의 사용되지 않은 과거 바이너리 객체를 스캔합니다. 사용자는 사용되지 않는 파일을 일괄 삭제하여 로컬 디스크 공간을 확보할 수 있습니다.
• 자동 동기화 (Auto Sync) (체크박스)
  - 자동 동기화를 전환합니다. 체크(ON)하면 애플리케이션 시작 시 또는 작업공간/저장소가 변경될 때 "최신 버전 가져오기 (Get Latest Version (Sync))" 후 "main 브랜치를 현재 브랜치에 병합 (Merge main branch into current branch)"을 순차적으로 자동 실행합니다.
• 라이브 모니터 (Live Monitor) (카드 및 상태 패널)
  - SolidWorks 및 저장소 메트릭의 실시간 상태를 표시합니다:
    - SolidWorks 상태 (SolidWorks Status): SolidWorks가 현재 실행 중인지 표시합니다(활성(Active) 또는 비활성(Inactive)).
    - 총 파일 수 (Total Files): 작업공간에서 버전 추적 중인 파일의 총 개수(무시된 경로 제외).
    - 열린 파일 (Open Files): 이 프로젝트에서 현재 SolidWorks로 열려 있는 파일 수.
    - 잠긴 파일 (Locked Files): Git LFS에서 잠긴 파일 수.
    - 저장소 크기 (Repository Size): Git 저장소 디렉토리 내에서 추적되는 모든 파일의 총 물리적 크기(자동 계산, B, KB, MB, GB 단위).

----------------------------------------------------------
2. 파일 관리자 모드 (FILE MANAGER MODE)
----------------------------------------------------------
파일 관리자(File Manager)는 파일 수준의 체크아웃, 체크인 및 로컬 수정 관리 기능을 제공합니다.

• 파일 테이블 (File Table) (트리뷰 테이블)
  - 작업공간의 모든 파일을 다음 열과 함께 나열합니다:
    - 파일 경로 (File Path): 저장소 기준 상대 경로. 행은 색상으로 구분됩니다: 파트(.sldprt)는 녹색(Green), 어셈블리(.sldasm)는 주황색(Orange), 도면(.slddrw)은 빨간색(Red), 기타 파일은 보라색(Purple)입니다.
    - 상태 (Status): 파일이 수정되지 않음(Unmodified), 로컬에서 수정됨(Modified), 또는 새 파일(New File)인지 표시합니다.
    - SolidWorks (SolidWorks): 파일이 현재 CAD 애플리케이션에 로드되어 있으면 'Open'을 표시합니다.
    - 잠김 (Locked): 잠금 상태를 나타냅니다('me'는 사용자가 잠금, 'other'는 다른 사용자가 잠금).
    - 소유자 (By): 잠금을 보유한 개발자 이름을 표시합니다.
  - 키보드 단축키(Keyboard Shortcut): 파일 테이블에 포커스가 있을 때 Ctrl+A, Ctrl+a 또는 Ctrl+ㅁ을 누르면 목록의 모든 파일이 자동 선택됩니다.
  - 정렬 옵션 (Sorting Options) (콤보박스): 이름별(by Name), 확장자별(by Extension), 상태별(by Status), SolidWorks별(by Solidworks) 또는 잠금별(by Locked) 중에서 선택합니다.
    - '상태별(by Status)'은 새 파일(New File) -> 수정됨(Modified) -> 수정되지 않음(Unmodified) 순으로 파일을 그룹화합니다.
    - 'SolidWorks별(by Solidworks)'은 'Open' 상태의 파일을 먼저 표시합니다.
    - 모든 정렬 모드는 전체 상대 파일 경로(File Path)를 보조 알파벳순 정렬 키로 사용합니다.
    - .gitignore에 정의된 무시된 파일(git ls-files로 감지)은 필터링되어 표시되지 않습니다.
• CAD 파일 미리보기 캔버스 (CAD File Preview Canvas) (사이드바)
  - 선택한 SolidWorks 파일(.sldprt, .sldasm, .slddrw)의 4:3 비율(180x135) 썸네일 이미지를 사이드바에 표시합니다. 이 미리보기는 OLE 파싱과 Windows Shell COM 인터페이스를 혼합하여 썸네일을 자동으로 추출하므로 SolidWorks 2026 이상 형식과 호환됩니다.
  - 미리보기는 파일 관리자 모드(File Manager)에서만 활성화되며 CAD 파일이 정확히 하나 선택된 경우에만 작동합니다.
• 클릭하여 복사 (Click-to-Copy) (캔버스 상호작용)
  - CAD 미리보기 캔버스를 클릭하면(마우스 커서가 손 모양으로 변경) 썸네일의 고해상도 비트맵 데이터가 Windows 시스템 클립보드(CF_DIB)로 복사됩니다. 이렇게 하면 Ctrl+V를 사용하여 PowerPoint, Word, 그림판(Paint) 또는 채팅 앱에 즉시 이미지를 붙여넣을 수 있습니다.
• 새로고침 (Refresh) (버튼)
  - 파일을 다시 로드하고, LFS 잠금 데이터베이스를 조회하며, SolidWorks 열린 문서를 즉시 업데이트합니다.
• 최상위 찾기 (Find Top) (버튼)
  - SolidWorks의 GetDocumentDependencies2 COM API를 사용하여 작업공간의 .sldasm 종속성 그래프를 스캔합니다.
  - 파일을 열지 않고 어셈블리 종속성 메타데이터를 읽어 기존 OpenDoc6 방식보다 훨씬 빠르게 스캔합니다.
  - 식별된 최상위 어셈블리(top-level assemblies)(작업공간 내 다른 .sldasm에서 참조되지 않는 파일)는 파일 테이블에 빨간색 굵은 "TOP" 레이블로 표시됩니다.
  - 백그라운드 하위 프로세스(sw_top_finder.py)에서 실시간 진행률 표시와 함께 실행됩니다.
• 열기 (Open) (버튼)
  - 현재 작업공간 폴더를 Windows 파일 탐색기(File Explorer)로 엽니다.
• 잠금 (Lock) (버튼)
  - 선택한 파일에 LFS 잠금을 획득하여 다른 설계자가 작업을 덮어쓰지 못하도록 합니다. (SolidWorks에서 파일을 열 때도 자동으로 잠금이 획득됩니다.)
• 잠금 해제 (Unlock File) (버튼)
  - 선택한 파일의 LFS 잠금을 수동으로 해제합니다.
• 강제 잠금 해제 (Force Unlock) (버튼)
  - 다른 개발자의 LFS 잠금을 선택한 파일에서 강제로 해제합니다. (주의해서 사용하세요!)
• 폐기 (Discard) (버튼)
  - 선택한 파일의 모든 로컬 수정사항을 취소하여 마지막 커밋 상태로 되돌립니다.
• eDrawings (버튼)
  - 선택한 CAD 파일을 eDrawings 애플리케이션으로 열어 빠르고 가볍게 봅니다.
• SolidWorks (버튼)
  - 선택한 CAD 파일을 활성 SolidWorks 인스턴스에서 엽니다.
• 비교 (Diff) (버튼)
  - 단일 파트(.sldprt) 또는 도면(.slddrw) 파일이 선택되고 백그라운드 작업이 실행 중이 아닐 때만 활성화됩니다.
  - 선택한 파일의 커밋 내역을 표시하는 팝업을 엽니다. 커밋을 선택하고 비교(Diff) 버튼을 클릭하면 현재 버전을 `_OURS`로 복사하고 선택한 커밋 버전을 `_THEIRS`로 추출하여(Git LFS smudge 지원) `.backup/` 디렉토리에 저장합니다.
  - 두 파일이 모두 SolidWorks에서 열립니다(SolidWorks가 실행 중이 아니면 자동 실행). 사용자가 SolidWorks 메뉴에서 수동으로 비교를 실행하도록 안내하는 정보 팝업이 표시됩니다:
    - 파트(.sldprt): "도구 > 비교 > 형상 메뉴를 사용하여 비교하십시오. (Use Tools > Compare > Geometry menu to compare.)"
    - 도면(.slddrw): "도구 > 비교 > 도면 비교 메뉴를 사용하여 비교하십시오. (Use Tools > Compare > Draw Compare menu to compare.)"
  - 두 파일은 사용자가 검사할 수 있도록 SolidWorks에 열린 상태로 유지되며, 문서가 닫히거나 SolidWorks가 종료되지 않습니다.
• 내보내기 (EXPORT) (버튼)
  - 도면(.slddrw), 파트(.sldprt) 및 어셈블리(.sldasm)를 다양한 형식(PDF, DXF, STEP, STEP_ASM)으로 일괄 변환하는 간편한 대화상자를 실행합니다.
  - 다중 형식 선택, 사용자 정의 파일 이름 접두사 필터(PREFIX) 및 대상 하위 디렉토리 설정(OUTPUT_DIR, 기본값 "2D")을 지원합니다.
  - 대화상자는 별도의 백그라운드 Python 프로세스에서 완전히 비동기적으로 실행되므로 메인 클라이언트 인터페이스는 계속 응답 가능합니다.
  - 대화상자 내에서 **INFO** 버튼을 클릭하면 요약 개수와 활성 내보내기 규칙과 일치하는 파일의 색상 구분 상대 경로 목록(확장자별 정렬)을 표시하는 상세 모달이 열립니다. 창은 목록과 "닫기(Close)" 버튼을 완전히 표시하도록 동적으로 크기가 조정됩니다(600x600).
  - 변환 품질 및 안전 기능 (Conversion Quality & Safety Features):
    - 도면을 PDF로 저장할 때 흑백(Black & White, swPDFExportInColor를 False로 설정) 고품질(High Quality) 렌더링이 강제 적용되며, SolidWorks 문서 속성의 사용자 정의 선 가중치/펜 테이블이 자동으로 적용됩니다(swPDFExportUseCurrentPrintLineWeights를 True로 설정).
    - 파트와 어셈블리를 STEP으로 저장할 때 AP214 표준(색상 지원)이 강제 적용되며, 색상 모양과 텍스처가 보존됩니다(swStepExportAppearances를 True로 설정). 모델에 2개 이상의 구성(Configurations)이 있는 경우 각 구성에 대해 별도의 STEP 파일이 자동 생성됩니다(예: filename_ConfigName.step).
    - 일괄 변환이 완료되면 원래 SolidWorks 사용자 기본 설정이 자동으로 복원됩니다.
    - **정확한 진행 개수 (Accurate Progress Count)**: "완료(Completed)" 카운터는 생성된 STEP 출력 또는 구성별이 아닌 처리된 CAD 소스 파일당 정확히 1씩 증가하므로, 여러 구성이 있는 모델에서도 정확한 개수가 보장됩니다.
    - **감시 타이머 보호 (Watchdog Protection)**: 파일 변환이 중단되거나 SolidWorks 교착 상태가 3분(180초) 이상 지속되면 감시 타이머(watchdog timer)가 중단된 프로세스와 SolidWorks 엔진을 종료하고, 새 인스턴스를 실행한 후 목록의 다음 파일부터 자동으로 변환을 재개합니다. SolidWorks가 제시간에 다시 시작되지 않으면 루프는 나머지 파일을 계속 시도합니다(조기 중단하지 않음).
    - **강력한 유니코드 및 UTF-8 인코딩 (Robust Unicode & UTF-8 Encoding)**: 모든 통신 파이프라인에 UTF-8 표준 스트림 재구성 및 오류 대체 처리(errors="replace")를 적용합니다. 이는 지역화된/비ASCII 구성 이름(예: `기본`)을 출력할 때 발생하는 Tkinter UI 진행 모니터 스레드 충돌을 완전히 방지합니다.
    - **교착 상태 없는 CloseDoc 처리 (Deadlock-Free CloseDoc Handling)**: 절대 파일 경로 대신 문서 제목(Title, GetTitle()로 검색)으로 문서를 닫습니다. CloseDoc을 호출하기 직전에 모든 활성 Python COM 래퍼(model = None, docs_left = None 등)를 해제하고 가비지 컬렉션(gc.collect() 및 CoCollectFreeUnusedLibraries())을 강제 실행하여 SolidWorks가 참조 잠금 없이 문서 리소스를 안전하게 언로드하도록 합니다.
• 버전 설명 (Version Description) (텍스트 상자)
  - 파일에 수행한 변경사항에 대한 간단한 설명(커밋 메시지)을 입력합니다.
• 선택 파일 버전 업로드 (Upload Selected File Version) (버튼)
  - 선택한 수정된 파일을 커밋, 푸시 및 업로드하고, 성공 시 잠금을 해제합니다.
  - 안전 검사(Safety Checks): 선택한 파일이 SolidWorks에서 현재 열려 있으면 영어 경고 대화상자와 함께 업로드가 차단됩니다. 다른 개발자가 파일을 잠근 경우 해당 파일을 건너뛰고 계속 진행하거나 업로드를 중단할 수 있습니다.
• 모든 파일 버전 업로드 (Upload Every Files Version) (버튼)
  - 작업공간의 모든 수정된 파일을 커밋, 푸시 및 업로드하고, 모든 활성 잠금을 해제합니다.
  - 안전 검사(Safety Checks): 선택 업로드와 동일한 SolidWorks 열린 파일 차단 및 다른 사용자 잠금 확인 검사를 수행합니다.

----------------------------------------------------------
3. 히스토리 로그 모드 (HISTORY LOG MODE)
----------------------------------------------------------
히스토리 로그(History Log)를 사용하면 이전 버전을 탐색하고 수정 내역을 감사할 수 있습니다.

• 리비전 목록 (Revision List) (테이블)
  - 활성 브랜치의 모든 과거 커밋 로그(커밋 해시(Commit Hash), 작성자(Author), 날짜(Date), 메시지(Message))를 표시합니다. 현재 체크아웃 버전의 행은 굵은 글씨(Bold)와 테마 녹색(#059669)으로 강조 표시됩니다.
• 그래프 (Graph) (버튼)
  - 화면 오른쪽 하단에 있습니다. 클릭하면 별도의 Windows 명령 터미널이 열리고 사용자 정의 git 실행 파일 경로를 사용하여 "git log --graph --oneline --all --decorate"로 ASCII 커밋 트리가 표시됩니다.

----------------------------------------------------------
4. 관리자 모드 (MAINTAINER MODE)
----------------------------------------------------------
프로젝트 관리자/유지보수자(Maintainer)를 위한 전용 모드로, 통합을 감독하고 새 저장소를 초기화합니다.

• 모든 브랜치를 main에 병합 (Merge all branches into main) (버튼)
  - 모든 원격 브랜치를 가져오고(fetch), 로컬로 풀(pull)하고, 선택에 따라 충돌을 해결한 후, 모두 'main'에 병합하고, 업데이트된 'main'을 원격 origin에 푸시하고, 저장소를 원래 개발자 브랜치로 되돌립니다.
• 저장소 이름 (Repository Name) (입력) 및 새 CAD 저장소 만들기 (Create New CAD Repository) (버튼)
  - 저장소 이름을 입력하고 "만들기(Make)"를 클릭하면 GitHub에 비공개 저장소가 자동으로 생성되고, 로컬로 초기화되며, Git LFS 구성이 적용되고, 잠금 파일 확장자(.sldprt, .sldasm, .slddrw)가 할당되고, 원격 추적이 설정되고, 초기 커밋이 푸시됩니다. 성공적으로 생성되면 클라이언트가 자동으로 대시보드(Dashboard) 작업공간 구성을 업데이트하고 대시보드 보기(Dashboard view)로 리디렉션합니다.

----------------------------------------------------------
5. 설정 모드 (CONFIG MODE)
----------------------------------------------------------
시스템 및 애플리케이션 경로를 구성하여 통합이 원활하게 실행되도록 합니다.

• 경로 설정 (Path Configurations) (입력 필드)
  - Git, Git LFS, SolidWorks, eDrawings, Git 토큰(Git Token), Git 서버 유형(Git Server Type), Git 서버 URL(Git Server URL), 기본 로컬 디렉토리(Default Local Directory) 및 기본 GitHub 조직 이름(Default GitHub Organization Name)에 대한 사용자 정의 실행 파일/폴더 경로입니다. 각 경로 필드(Git, Git LFS, SolidWorks, eDrawings)에는 콤보박스와 C 드라이브에서 해당 실행 파일을 스캔하여 드롭다운 목록에서 경로를 선택할 수 있는 "찾기(Find)" 버튼이 있습니다. "Git 서버 유형(Git Server Type)" 콤보는 "Git 서버 URL(Git Server URL)" 입력을 전환합니다(github→비활성화(disabled), gitea→활성화(enabled)). "기본 로컬 경로(Default Local Path)" 필드에는 디렉토리 선택기를 여는 "찾기(Find)" 버튼이 있습니다. 참고: "auto_sync" 구성 매개변수는 config.json에 부울(true/false)로 저장되어 자동 동기화를 활성화 또는 비활성화합니다. 이는 대시보드 모드(Dashboard Mode)의 자동 동기화(Auto Sync) 체크박스를 통해 관리되며 설정 모드(Config Mode)에서 수동 편집에서 제외됩니다.
• 설정 저장 (Save Configuration) (버튼)
  - 모든 설정 값을 config.json에 저장하고 Git 및 COM 연결을 다시 초기화합니다.
• 편집 (Edit) (버튼)
  - 현재 설정 파일(--config 인수로 지정, 기본값 config.json)을 GUI 양식을 거치지 않고 메모장(Notepad)에서 직접 열어 수동 편집합니다.

----------------------------------------------------------
6. 시스템 로그 및 프로세스 종료 (SYSTEM LOG & PROCESS TERMINATION)
----------------------------------------------------------
클라이언트 창 하단에는 시스템 로그(System Log)가 진행 중인 작업에 대한 정보(Info), 성공(Success), 경고(Warning) 또는 오류(Error) 로그를 표시합니다.

• 상태 표시기 (Status Indicator)
  - 백그라운드 작업이 실행 중이 아닐 때는 녹색 점과 "● 대기 중 (Idle)"을, 작업이 활성화되어 있을 때는 빨간색 점과 "● 작업 중 (Working)"을 표시합니다.
• 순차 버튼 대기열 (Sequential Button Queuing)
  - 백그라운드 작업이 실행 중(Working 상태)일 때 작업 버튼(예: 병합(Merge), 동기화(Sync), 업로드(Upload))을 클릭하면 해당 작업이 self.pending_button_tasks에 대기열에 추가되고 활성 프로세스가 완료되면 순차적으로 자동 실행됩니다.
• 종료 (Terminate) (버튼)
  - 진행 중인 백그라운드 Git 작업을 강제로 중단하고 종료할 수 있습니다. Windows에서 생성된 모든 하위 프로세스 트리를 재귀적으로 종료하고, 대기 중인 버튼 작업을 모두 지우고, 클라이언트 버튼을 기본 활성 상태로 복원합니다.
  - 상태가 "대기 중(Idle)"일 때는 버튼이 시각적으로 비활성화(연한 분홍색)되고, 백그라운드 작업이 실행 중일 때만 완전히 활성화(빨간색 배경)됩니다.
• 지우기 (Clear) (버튼)
  - 현재 텍스트 로그 창을 비웁니다.
