GIT4SW - SolidWorks Git Version Control Client User Guide
==========================================================

GIT4SW is an agentic CAD version control desktop application designed for SolidWorks designers. It integrates standard Git workflows with Git LFS (Large File Storage) locking mechanisms to enable concurrent, overwrite-free collaboration on 3D CAD models.

Below is a detailed walkthrough of each application tab, its widgets, and functions.

----------------------------------------------------------
1. DASHBOARD MODE
----------------------------------------------------------
The Dashboard is the central cockpit of the application, showing repository configurations, sync controls, and real-time statuses.

• Local Path (Text Field & Entry)
  - Displays the absolute path to your active local Git repository.
• Change Workspace (Button)
  - Opens a folder chooser dialog to switch the workspace to another Git repository directory.
• Remote Server (Text Field & Entry)
  - Displays the remote repository URL currently linked to the workspace, or allows you to enter a URL to clone a new repository.
• Clone (Button)
  - Clones the Git repository from the Remote Server URL into the directory specified in Local Path.
• Active Branch (Combobox/Dropdown)
  - Displays the currently checked-out branch. Selecting a different branch from this dropdown automatically switches (checkouts) the workspace to that branch.
• Make my branch (Button)
  - Resolves your identity from your git/GitHub configuration, automatically creates a new local and remote developer branch under your username, and switches to it. This keeps your work isolated and prevents direct modification conflicts on the 'main' branch.
• README.md (Button)
  - Opens the README.md file of your active local repository in Notepad. Once you edit and close the text editor, the client automatically runs git add, commit, and push to synchronize the file with the remote server. If the file does not exist locally, it initializes it from the template.
• Get Latest Version (Sync) (Button)
  - Downloads and applies (pulls) the latest changes from the remote server for the active branch.
• Merge main branch into current branch (Button)
  - Merges updates from the shared 'main' (or 'master') branch into your developer branch. It automatically commits, pushes the merged branch back to the remote server, and safely switches the workspace back to your original developer branch.
• Auto Sync (Checkbox)
  - Toggles automated synchronization. When checked (ON), the application automatically executes "Get Latest Version (Sync)" followed by "Merge main branch into current branch" sequentially whenever the application starts or the workspace/repository is changed.
• Live Monitor (Card & Status Panel)
  - Displays real-time status of SolidWorks and repository metrics:
    - SolidWorks Status: Shows whether SolidWorks is currently running (Active or Inactive).
    - Total Files: The total count of version-tracked files in the workspace (excluding ignored paths).
    - Open Files: Number of files from this project currently open in SolidWorks.
    - Locked Files: Number of files locked under Git LFS.

----------------------------------------------------------
2. FILE MANAGER MODE
----------------------------------------------------------
The File Manager provides file-level checkout, checking-in, and local modifications management.

• File Table (Treeview Table)
  - Lists all files in the workspace with the following columns:
    - File Path: Path relative to repository. Rows are color-coded: Parts (.sldprt) are Green, Assemblies (.sldasm) are Orange, Drawings (.slddrw) are Red, and other files are Purple.
    - Status: Shows whether a file is unmodified, modified locally (Modified), or a new file (New File).
    - SolidWorks: Displays 'Open' if the file is currently loaded in the CAD application.
    - Locked: Indicates lock state ('me' if locked by you, 'other' if locked by someone else).
    - By: Displays the name of the developer who holds the lock.
  - Keyboard Shortcut: Pressing Ctrl+A, Ctrl+a, or Ctrl+ㅁ when focusing the file table automatically selects all files in the list.
  - Sorting Options (Combobox): Choose from by Name, by Extension, by Status, by Solidworks, or by Locked.
    - 'by Status' groups files by New File -> Modified -> Unmodified.
    - 'by Solidworks' groups files with 'Open' status first.
    - All sorting modes utilize the full relative File Path as the secondary alphabetical sorting key.
    - Ignored files defined in .gitignore (detected via git ls-files) are filtered out and will not be displayed.
• CAD File Preview Canvas (Sidebar)
  - Displays a 4:3 ratio (180x135) thumbnail image of the selected SolidWorks file (.sldprt, .sldasm, .slddrw) in the sidebar. This preview automatically extracts thumbnails using a hybrid of OLE parsing and Windows Shell COM interfaces, making it compatible with SolidWorks 2026 and newer formats.
  - The preview is active ONLY in File Manager mode and when exactly one CAD file is selected.
• Click-to-Copy (Canvas Interaction)
  - Clicking the CAD preview canvas (mouse cursor changes to a hand shape) copies the high-resolution bitmap data of the thumbnail to the Windows system clipboard (CF_DIB). This allows you to immediately paste the image into PowerPoint, Word, Paint, or chat apps using Ctrl+V.
• Refresh (Button)
  - Reloads files, queries LFS lock database, and updates SolidWorks open documents instantly.
• Open (Button)
  - Opens the current workspace folder in Windows File Explorer.
• Lock (Checkout) (Button)
  - Checks out the selected files and acquires LFS locks on them to prevent other designers from overwriting your work. (Locks are also automatically acquired when you open files in SolidWorks).
• Unlock File (Button)
  - Manually releases your LFS locks on the selected files.
• Force Unlock (Button)
  - Forcefully breaks another developer's LFS lock on the selected files. (Use with caution!).
• Discard (Button)
  - Undoes all local modifications to the selected files, restoring them to the last commit.
• eDrawings (Button)
  - Opens the selected CAD file in the eDrawings application for quick, lightweight viewing.
• Solidworks (Button)
  - Opens the selected CAD file in the active instance of SolidWorks.
• Version Description (Text Box)
  - Enter a brief description (commit message) describing what changes you made to the files.
• Upload Selected File Version (Button)
  - Commits, pushes, and uploads selected modified files to the remote server, releasing locks upon success.
  - Safety Checks: If any selected file is currently open in SolidWorks, the upload is blocked with an English warning dialog. If any file is locked by another developer, you will be prompted to skip those files and proceed, or abort the upload.
• Upload Every Files Version (Button)
  - Commits, pushes, and uploads all modified files in the workspace, and releases all active locks.
  - Safety Checks: Performs the same SolidWorks open file blocks and other-user lock validation checks as the selected upload.

----------------------------------------------------------
3. HISTORY LOG MODE
----------------------------------------------------------
The History Log lets you browse previous revisions and audit modifications.

• Revision List (Table)
  - Shows a log of all past commits (Commit Hash, Author, Date, Message) for the active branch. The current checkout version's row is highlighted in Bold text and themed Green color (#059669).
• Graph (Button)
  - Located at the bottom right corner of the screen. Clicking it opens a separate Windows command terminal displaying the ASCII-drawn commit tree using "git log --graph --oneline --all --decorate", matching your custom git executable path.

----------------------------------------------------------
4. MAINTAINER MODE
----------------------------------------------------------
Dedicated to project administrators/maintainers to oversee integration and initialize new repositories.

• Merge all branches into main (Button)
  - Fetches all remote branches, pulls them locally, resolves conflicts according to your choices, merges them all into 'main', pushes the updated 'main' back to remote origin, and returns the repository back to the original developer branch.
• Repository Name (Entry) & Create New CAD Repository (Button)
  - Enter a repository name and click "Make" to automatically create a private repository on GitHub, initialize it locally, apply Git LFS configuration, assign locked file extensions (.sldprt, .sldasm, .slddrw), set up remote tracking, and push the initial commit. Upon successful creation, the client automatically updates the Dashboard workspace configuration and redirects you to the Dashboard view.

----------------------------------------------------------
5. CONFIG MODE
----------------------------------------------------------
Configure system and application paths to ensure integration runs smoothly.

• Path Configurations (Entries)
  - Custom executable/folder paths for Git, Git LFS, SolidWorks, eDrawings, GitHub Personal Access Token, default local directory, and default GitHub organization name. Note: The "auto_sync" configuration parameter is saved in config.json as a boolean (true/false) to enable or disable automatic synchronization. It is managed via the Auto Sync checkbox in Dashboard Mode and is excluded from manual editing in Config Mode.
• Save Configuration (Button)
  - Saves all configured values to config.json and re-initializes Git and COM connections.

----------------------------------------------------------
6. SYSTEM LOG & PROCESS TERMINATION
----------------------------------------------------------
At the bottom of the client window, the System Log displays info, success, warning, or error logs about ongoing operations.

• Status Indicator
  - Displays a green dot and "● Idle" when no background tasks are running, and a red dot and "● Working" when operations are active.
• Sequential Button Queuing
  - If you click any action button (e.g. merge, sync, upload) while a background task is running (Working state), the action is queued in self.pending_button_tasks and will automatically execute sequentially once the active process finishes.
• Terminate (Button)
  - Allows you to forcefully abort and terminate any ongoing background Git operations. It recursively terminates all spawned child subprocess trees on Windows, clears any queued pending button tasks, and restores the client buttons to their default active states.
  - The button is visually disabled (faded pink color) when the status is "Idle" and turns fully active (red background) only when background tasks are running.
• Clear (Button)
  - Empties the current text log window.
